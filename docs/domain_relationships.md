# Seesaw 도메인 엔터티 관계

이 문서는 Seesaw 프로젝트의 도메인 엔터티 간의 관계를 설명합니다.

## 개요

도메인 모델은 `Site`, `Category`, `Article`, `User`를 핵심 엔터티로 구성하며, `AbstractHierarchical`을 통해 계층 구조를 지원합니다. 접근 제어는 `Role`, `RoleMapping`, `Permission`을 통해 관리됩니다.

## 엔터티 관계

### 1. Site (사이트)
*   **자기 참조 (Self-Referencing)**: 사이트는 부모 사이트와 여러 자식 사이트를 가질 수 있습니다 (계층형).
*   **Category와 일대다 관계**: 하나의 사이트는 여러 카테고리를 가질 수 있습니다 (`0..N`). 카테고리는 정확히 하나의 사이트에 속합니다.
*   **RoleMapping과 일대다 관계**: 하나의 사이트는 여러 역할 매핑을 가질 수 있습니다 (`0..N`). 이는 특정 사이트에서 어떤 사용자가 어떤 역할을 갖는지를 정의합니다.

### 2. Category (카테고리)
*   **자기 참조 (Self-Referencing)**: 카테고리는 부모 카테고리와 여러 자식 카테고리를 가질 수 있습니다 (계층형).
*   **Site와 다대일 관계**: 카테고리는 하나의 사이트에 속합니다. 사이트 없이는 존재할 수 없습니다.
*   **Article과 일대다 관계**: 하나의 카테고리는 여러 게시글을 포함할 수 있습니다 (`0..N`). 게시글은 정확히 하나의 카테고리에 속합니다.

### 3. Article (게시글)
*   **자기 참조 (Self-Referencing)**: 게시글은 부모 게시글과 여러 자식 게시글을 가질 수 있습니다 (계층형).
*   **Category와 다대일 관계**: 게시글은 하나의 카테고리에 속합니다. 카테고리 없이는 존재할 수 없습니다.
*   **Reply와 일대다 관계**: 하나의 게시글은 여러 댓글을 가질 수 있습니다 (`0..N`).
*   **View와 일대다 관계**: 하나의 게시글은 여러 조회 기록(View)을 가질 수 있습니다 (`0..N`).

### 4. Reply (댓글)
*   **자기 참조 (Self-Referencing)**: 댓글은 부모 댓글(대댓글)과 여러 자식 댓글을 가질 수 있습니다.
*   **Article과 다대일 관계**: 댓글은 하나의 게시글에 속합니다. 게시글 없이는 존재할 수 없습니다.

### 5. User (사용자)
*   **RoleMapping과 일대다 관계**: 사용자는 여러 역할 매핑을 가질 수 있습니다 (`0..N`). 이는 사용자를 다양한 사이트의 여러 역할과 연결합니다.

### 6. Role (역할)
*   **RoleMapping과 일대다 관계**: 역할은 역할 매핑을 통해 여러 사용자/사이트에 할당될 수 있습니다 (`0..N`).
*   **MenuRole과 일대다 관계**: 역할은 메뉴 역할(MenuRole)을 통해 여러 메뉴에 매핑될 수 있습니다 (`0..N`).

### 7. RoleMapping (역할 매핑)
*   **User와 다대일 관계**: 특정 사용자와 연결됩니다.
*   **Site와 다대일 관계**: 특정 사이트와 연결됩니다.
*   **Role과 다대일 관계**: 특정 역할과 연결됩니다.
*   *설명*: 이 엔터티는 User, Site, Role 간의 다대다 관계를 해소합니다. "사용자 U는 사이트 S에서 역할 R을 가진다"를 정의합니다.

### 8. Menu (메뉴)
*   **자기 참조 (Self-Referencing)**: 메뉴는 부모 메뉴와 여러 자식 메뉴를 가질 수 있습니다 (계층형).
*   **MenuRole과 일대다 관계**: 하나의 메뉴는 여러 메뉴 역할(MenuRole)을 가질 수 있습니다 (`0..N`).

### 9. MenuRole (메뉴 역할)
*   **Menu와 다대일 관계**: 특정 메뉴와 연결됩니다.
*   **Role와 다대일 관계**: 특정 역할과 연결됩니다.
*   *설명*: 이 엔터티는 Menu와 Role 간의 다대다 관계를 해소합니다. 어떤 역할이 어떤 메뉴에 접근할 수 있는지를 정의합니다.

### 10. View (조회)
*   **Article과 다대일 관계**: 조회 기록은 하나의 게시글에 속합니다.

### 11. Code (코드)
*   **자기 참조 (Self-Referencing)**: 코드는 부모 코드와 여러 자식 코드를 가질 수 있습니다 (계층형).

### 12. 논리적 관계 (직접적인 JPA 매핑 없음)
일부 엔터티는 직접적인 객체 매핑 대신 ID 참조를 통해 관계를 유지합니다:

*   **Attachment (첨부파일)**:
    *   `referenceId` 필드를 가집니다. 이는 `Type` (PROFILE, BACKGROUND_IMAGE, INLINE_IMAGE, ATTACHMENT)에 따라 다양한 엔터티(예: Article, User Profile)와 연결될 수 있습니다.
*   **Permission (권한)**:
    *   `targetId`와 `roleId`를 가집니다. 이는 특정 대상 객체(ID)를 역할(Role)에 연결하여 접근 제어 목록(ACL)을 정의합니다.
*   **Vote (투표)**:
    *   `referenceId` 필드를 가집니다. 이는 투표를 대상 엔터티(예: Article, Reply)와 연결합니다.

## 상속 구조

*   **AbstractHierarchical**: 다음 엔터티들은 이 클래스를 상속받아 계층(트리) 구조를 지원합니다:
    *   `Site`
    *   `Category`
    *   `Article`
    *   `Reply`
    *   `Menu`
    *   `Code`
    *   *참고*: 이 클래스는 `parent`와 `children` 필드를 제공합니다.

*   **BaseEntity**: 모든 엔터티는 `BaseEntity`를 상속받으며(직접 또는 `AbstractHierarchical`을 통해), `id`, `createdBy`, `createdDate`, `lastModifiedBy`, `lastModifiedDate` 등의 공통 필드를 제공합니다.
